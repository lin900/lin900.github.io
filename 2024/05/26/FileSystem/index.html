FileSystem

Created2024-05-26|Updated2024-06-02
|Post Views:

文件系统

块设备

传统机械式硬盘：磁盘内有多个轨道，探头一次读取一块数据，单字节读取是不可能的
固态硬盘（SSD，Solid State Disk）：阵列式存储，一次读取一行

基于indoe的文件系统

indoe是常规文件的数据索引方式
48K+6M+1G

基于表的文件系统

FAT:文件分配表(File Allocation Table)

Windows系统使用至今
数据基本块单位为簇，4K大小

FAT是簇序号的数组，表示下一个簇，文件由簇的链表形式组织

链表容易断（硬盘坏道），创建了一模一样的FAT2当备份

遍历链表，文件越大，读写越慢

FAT32中，文件大小元数据使用32位整数表示，文件最大为4G

exFAT:U盘常用格式

  • 与FAT32不兼容
  • 使用位图加快空间分配
  • Unicode保存长文件名
  • 目录查找文件使用哈希对比
  • 允许4G以上文件（文件大小使用8字节）
  • 使用校验码保证元数据完整性

NTFS:NEW Technology File System

现代Windows系统中广泛使用的文件系统
核心结构主文件表（Master File Table，MFT）

前16个保留记录为元数据，记录基本信息
$MFT自举：存储自身数据
MFT为一个B+树结构
